Now you are required to use the TypeScript language to implement a bargaining function component used by Texas Poker. The design document is as follows.
1. 界面设计细节：
1.1 外观设计：
1.1.1 形状、颜色、图案：
形状：
筹码采用圆形设计，直径为50像素，确保在不同屏幕上保持一致性。
颜色：
主色调为深蓝色，符合赌场主题。不同面额的筹码采用不同的辅助颜色，如红色、绿色、黑色等。
图案：
筹码中央显示扑克牌花色图标，四种花色分别为红桃、方块、梅花和黑桃。数字或面额以清晰的白色字体显示在图标旁边。
1.1.2 材质和质感：
材质：
通过阴影和渐变效果模拟陶瓷质感，使筹码看起来更加立体。
质感：
保持平滑的表面，以确保用户在搬动和投注时有舒适的触感。
1.1.3 尺寸和重量：
尺寸：
确保筹码的标准尺寸，直径50像素，高度约为10像素。
重量：
虚拟筹码不涉及实际重量，但动画效果和用户体验应该使搬动和投注感觉逼真而自然。
1.1.4 图标和符号：
图标设计：
扑克牌花色图标采用清晰的矢量图形，确保在不同分辨率下显示清晰。
数字和面额显示：
数字和面额采用Sans-serif字体，大小适中，确保用户可以轻松辨别。
1.1.5 定制选项：
定制外观：
提供用户可选的颜色主题，允许用户在一定范围内自定义筹码的颜色和图案，以增加个性化。
通过以上设计细节，确保筹码在外观上既符合赌场主题，又能够清晰传达面额和花色信息，使用户在游戏过程中有良好的视觉体验。
1. Interface design details:
1.1 appearance design:
1.1.1 shape, color, pattern:
shape:
The chip adopts a circular design with a diameter of 50 pixels to ensure that it is consistent on different screens.
color:
The main color tone is dark blue, which is in line with the theme of the casino. The chips of different surfaces use different auxiliary colors, such as red, green, black, etc.
pattern:
The center of the chip shows the poker color icon, and the four colors are red peaches, squares, plums and spades. The number or surface is displayed next to the icon with clear white fonts.
1.1.2 Material and texture:
Material:
The ceramic texture of the shadow and gradient effect makes the chips look more three -dimensional.
Texture:
Keep a smooth surface to ensure that users have a comfortable touch when moving and betting.
1.1.3 size and weight:
size:
Ensure the standard size of the chip, 50 pixels in diameter, and a height of about 10 pixels.
weight:
Virtual chips do not involve actual weight, but animation effects and user experiences should make moving and betting feels realistic and natural.
1.1.4 icon and symbol:
Icon design:
Poker colorful icons use clear vector graphics to ensure clear display at different resolution.
Display numerals and surfaces:
The numbers and surfaces are sans-server, which is moderate in size to ensure that users can easily distinguish.


2. 交互设计细节：
2.1 拖放和搬动：
2.1.1 拖放交互：
用户搬动筹码：
用户通过点击并拖动筹码，将其移动到所需的投注区域。
添加平滑的拖动动画，使搬动过程生动而自然。
2.1.2 搬动效果：
动画效果：
搬动时筹码应该在鼠标/触摸位置形成一个微小的阴影，模拟筹码离开桌面的感觉。
搬动过程中，筹码保持在最前层，以确保用户清晰地看到其位置。
2.2 投注和结算：
2.2.1 投注流程：
选择投注额：
用户通过点击筹码选择投注额，可以单击一次选择一个，或者通过拖动一次选择多个筹码。
结算动画：
当游戏进行结算时，获胜的投注区域的筹码应该以流畅的动画效果移到相应的用户账户区域。
2.3 实时更新：
2.3.1 实时变化：
WebSocket通信：
使用WebSocket技术与后端进行实时通信，确保筹码的变化反映游戏状态的实时变化。
动态更新：
在筹码上添加实时数字或标识，反映当前游戏状态，如总投注额、当前局面等。
2.4 用户反馈：
2.4.1 状态提示：
成功提示：
当用户成功搬动或投注时，通过弹出框或简短的消息提示反馈成功状态。
错误提示：
如果用户执行了不允许的操作，通过红色的闪烁或弹出框提示错误信息。
2.4.2 动画效果：
流畅动画：
在用户进行操作时，使用流畅的过渡动画来增强用户体验，确保界面变化不显得突兀。
2.5 定制选项：
2.5.1 用户定制：
颜色和图案：
提供颜色选择器和图案选项，以便用户在一定范围内自定义筹码的外观。
保存定制：
允许用户保存自定义的筹码外观，确保下次游戏时能够保留其个性化设置。
通过以上交互设计细节，确保用户在操作过程中能够顺畅地搬动、投注，并获得实时的游戏反馈，同时提供个性化的定制选项以增强用户参与感。

2. Interactive design details:
2.1 Drag and drop and move:
2.1.1 Drag and release interaction:
User moving chips:
The user moves it to the required betting area by clicking and dragging the chips.

Settlement animation:
When the game is settled, the chips of the winning betting area should be moved to the corresponding user account area with a smooth animation effect.
2.3 Real -time update:
2.3.1 Dynamic update:
Add real -time numbers or signs to the chips to reflect the current game status, such as the total betting amount, the current situation, etc.

Successful reminder:
When the user successfully moves or bets, the successful state is prompted by the pop -up box or a brief message.
Error message:
If the user executes non -allowable operations, an error message is prompted through the red flash or pop -up box.
2.4.2 Animation effect:
Smooth animation:
When operating by the user, use smooth transition animation to enhance the user experience and ensure that the interface changes are not obtrusive.
